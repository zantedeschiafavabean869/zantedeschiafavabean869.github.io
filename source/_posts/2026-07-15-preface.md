我是一个剑指游戏开发的软件工程专业学生，目前的学习方向主要是 Unity Client Dev，学习路线也比较清晰，不过由于我大一开学后 10 月份才正式接触编程，目前的 Dev 工作高度依赖 Claude Code 以及 Codex 这样的 Agent 工具辅助。

编程类书籍看过两三本，有基础的 C#、C、C++ 语法，数据结构没有系统学习，线性代数看了蛮多但没有运用，离散数学也只学了学校教授的内容。Coding 能力不强，大致的工程思维有，有 gitflow 工作流思维，目前也在轻度实践。

仿照教程抄过两个小游戏，基础的生命周期和回调函数以及基本的引擎使用都会。Agent 也帮我写过 Editor 工具，还蛮好用的。不过我的目标是成为一流的开发者，目前也对 LLM 这类 AI 工具在游戏创作上的应用感兴趣。Skill.md 和 MCP 我知道，但是不清楚，也没怎么用过，OpenClaw 我也没玩。

对于 Unreal Engine 5 这类重型工业化的引擎也比较感兴趣。我看过 .NET 的 Garbage Collection，知道有 3 代和 LOH，知道线程栈和托管堆，也知道对象池模式是为了解决堆上内存碎片化的问题，不过我不太会写对象池。设计模式我也就单例模式知道有饿汉和懒汉，不过用得也不熟。

我之前学 C++ 的时候也和 C# 进行了对比，比如 struct 和 class 在 C++ 中只有成员变量的默认可调用范围不同，而 C# 中一个是值类型一个是引用类型，性能开销不同。我知道商业游戏以性能为第一要素。计算机图形学方面，我知道线性代数的变换矩阵是为了计算下一帧的图像，也知道光栅化、网格这些概念。

最近在研究 3C，Cinemachine 比较好用，Virtual Camera 的设计理念我觉得很棒。我知道相机有正交相机和投影相机，我还去学了四元数。Unity 引擎的旋转顺序是 YXZ，而 Unreal Engine 5 的旋转顺序是 ZXY，三维的转动顺序是有影响的。

Control 方面，我知道 Unity 有两套输入系统。UI 方面我知道 UGUI 是怎么来的，也知道有 DOTween 这种好用的插件。我知道 Unity 有一种高性能的写法叫 DOTS，是一种 ECS，不过具体我没看懂，也没多看。物理系统方面，我知道很多都是用公式去模拟，比如移动的逻辑就是修改 transform 的 position 或是 rigidbody 的 velocity。我知道有一个东西叫布娃娃系统，不过我没用过。Unreal Engine 的 gameplay 我知道有 GAS。

我想梳理一下大二的学习顺序，也想认真考虑升学、实习这些路径。大二我还想带领游戏开发社团，教大一新生从基础开始搓小游戏，顺便补补我自己的基础。

我可能还有很多关于计算机基础、虚拟机、Linux、AI 相关的基础知识要学。
